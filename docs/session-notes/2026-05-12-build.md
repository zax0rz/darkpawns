## What Was Built

Three components of the memory system for the AIIDE 2027 paper, as specified in the kick-off brief from The Architect.

### 1. Content-Aware Valence Heuristics (`pkg/dreaming/extract.go`)

**Before:** Every attack = +1, every flee = -2, every acquisition = +1. Flat. Wrong.

**After:** Context-sensitive valence computation based on game state:

- **Kill valence:** Based on mob level relative to agent level.
  - Trivial (mob 10+ levels below): +0 (barely worth remembering)
  - Easy (3-10 below): +1
  - Challenging (within ±3 levels): +2
  - Epic (mob outlevels agent): +3

- **Flee valence:** Based on HP at time of retreat.
  - Fled at 80%+ HP: -3 (cowardly, something went wrong)
  - Fled at 40-80% HP: -2 (embarrassing but understandable)
  - Fled at 20-40% HP: -1 (tactical retreat)
  - Fled at <20% HP: 0 (survival instinct, not failure)

- **Social valence:** Based on interaction type keywords + simple speech sentiment.
  - Betrayal/backstab: -3
  - Gift: +2
  - Cooperation/heal: +1
  - Simple keyword sentiment on say lines (positive/negative word counts)

- **Acquisition valence:** Based on item level proxy (0-100 scale).
  - Legendary (80+): +3
  - Valuable (50-79): +2
  - Useful (20-49): +1

- **Damage valence:** Based on HP percentage when hurt.
  - Near-death (<5%): -3
  - Badly hurt (5-15%): -2
  - Moderate (15-30%): -1

**New fields in `LogEvent`:** `AgentLevel`, `MobLevel`, `SocialTarget`, `SocialType`, `ItemLevel`, `CombatResult`.

### 2. Narrative Summary Formatting (`pkg/dreaming/graph.go`)

**Before:** Flat bullet list with salience scores. `- Killed goblin in Dark Corridor [+2] (salience: 0.85)`

**After:** Prose narrative, chronologically ordered, grouped by session:

```
## Memory

### Session 1 — Jan 12 at 3:15 PM – 3:47 PM

Attacked goblins in the Dark Corridor (noteworthy).
Killed a rogue troll in the Mountain Pass (a significant moment).
Low HP (12/50) while fighting a cave bear (a difficult moment).

### Session 2 — Jan 12 at 4:02 PM

Picked up a gleaming sword in the Dragon's Lair (noteworthy).
Said "We should group up" in the Tavern.

### Relationships

Brenda — trusted ally (met 3 times)
Goblin Shaman — dangerous (met 2 times)
```

**Key design decisions:**
- Events sorted by `CreatedAt`, grouped by 30-minute time gaps into sessions
- High-salience events (>0.15) included; low-salience pruned silently
- Valence context appended as parenthetical: "(a significant moment)", "(a difficult moment)"
- Entity relationship summary at the end — accumulated valence per entity
- ~500 token budget, truncates with "(additional memories consolidated)"

### 3. Server-Side Memory Injection

**Files changed:**
- `pkg/session/memory_hooks.go` — new `SendMemorySummary()` method
- `pkg/session/manager.go` — new `dreamingDir` field + `SetDreamingDir()` setter
- `cmd/server/main.go` — wires `SetDreamingDir("data/dreaming")`
- `pkg/session/session_login.go` — calls `SendMemorySummary()` after agent auth
- `pkg/agentcli/client.go` — handles `memory_summary` message type, sets `state.MemorySummary`

**Flow:**
1. Dreaming layer writes `data/dreaming/{agent_id}/memory-summary.txt`
2. Agent authenticates → server calls `SendMemorySummary()`
3. Server reads the file, sends `{"type":"memory_summary","summary":"..."}` message
4. Client receives it, sets `a.state.MemorySummary`
5. LLM decision loop injects it as a system message

**Also cleaned up:** Removed dead `sortBySalience` and `formatNode` functions from graph.go (superseded by narrative formatting).
