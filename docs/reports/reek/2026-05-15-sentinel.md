## Notable (no action required)

### LOW: JSON string interpolation with err.Error()
**Files:** `pkg/admin/handlers.go` lines 528, 532, 646, 650, 907, 911, 1963
All seven sites construct JSON error responses via string concat: `{"error":"` + err.Error() + `"}`
Currently safe — all error sources are known literals (strconv, json.Decode, fmt.Errorf with no user input). But pattern is fragile: if any source introduces quotes or backslashes, JSON breaks. Prefer `json.NewEncoder` with a struct.

### MEDIUM-LOW: Redundant nested RLock in CalculateAC
**File:** `pkg/session/equipment_ac.go`
`CalculateAC` calls `p.RLock()` at function entry (deferred), then calls `p.RLock()` again mid-function. Works because `sync.RWMutex` permits nested read locks from the same goroutine, but redundant and confusing. The second lock/unlock pair can be removed — the outer defer covers the whole function.

### LOW: Route split creates fragile role boundary
**File:** `pkg/admin/router.go`
`handleZoneReset` (admin) at `/admin/zones/reset` and `handleZoneByIDOrReset` (builder) at `/admin/zones/` coexist via ServeMux longer-pattern priority. Correct today but fragile — any future routing refactor could accidentally drop the admin-only guard.
