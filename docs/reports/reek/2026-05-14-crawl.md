## HIGH

**engine/affect_tick.go:22** — TickManager default interval is 1 second, not 1 MUD hour (60s). The NewTickManager sets `tickInterval: time.Second`. If TickManager is started with defaults, affect durations tick down 60x faster than intended. A 10-tick poison that should last 10 minutes expires in 10 seconds. The comment says "Default: 1 tick per second" but the actual game loop ticks every 100ms and affect_update fires every 60s.
*Source: analysis — tick interval mismatch*
*Suggested fix:* change default to `60 * time.Second`

## MEDIUM

**engine/affect.go:173** — generateAffectID() uses `time.Now().Format("20060102150405")` + 8-char random for IDs. At 100ms tick rates with multiple affect applications per tick, collision risk exists despite the random suffix. The IDs are also never referenced externally, so the uniqueness guarantee is fake.
*Source: analysis*
*Suggested fix:* use a sync/atomic counter for IDs.
