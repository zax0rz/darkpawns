## CRITICAL

### Memory slice — concurrent read/write, no lock
**File:** `pkg/game/mobact.go:215`, `pkg/game/deferred_fight_fns.go:224,233`
**Source:** LLM analysis (race condition scan)
`ch.Memory` read in MobileActivity() AI tick without any mob-level lock. Remember() and Forget() write m.Memory without any lock. AI tick goroutine and combat goroutines append/iterate the same `[]string` pointer unsynchronized. In C this was single-threaded. In Go it's a classic data race.
**Fix:** Add m.mu RLock/Lock around all Memory accesses. Add GetMemory()/AddToMemory() accessors.

### Hunting/HuntingID data race
**File:** `pkg/game/deferred_fight_fns.go:199-206`
**Source:** LLM analysis (race condition scan)
m.Hunting/m.HuntingID written by SetHunting() without lock, read by GetHunting()/IsHunting() which take RLock. Write path skips the lock entirely.
**Fix:** Lock m.mu in SetHunting() before mutation.

### Package-level aiCombatEngine global — zero synchronization
**File:** `pkg/game/ai.go:31-33`, `pkg/game/mobact.go:91`
**Source:** LLM analysis (race condition scan)
var aiCombatEngine written by SetAICombatEngine() and read by every MobileActivity() iteration. No mutex, no atomic, no sync.
**Fix:** Wrap in atomic.Value or sync.RWMutex.

### executeMobCommand reads mob pointer after releasing world lock
**File:** `pkg/game/world.go:167-176`
**Source:** LLM analysis (concurrency)
Acquires w.mu.RLock() to find a mob by VNum, releases, then operates on dangling pointer. Mob could be extracted between RUnlock and use.
**Fix:** Keep world or mob lock across operation.

## Deep Dive Notes (Friday — World Entities)

2412 lines across mob.go, mobact.go, mobprogs.go, object.go, world_object.go, world_zone.go, zone_dispatcher.go.

The mob entity layer is the most faithfully ported C code — mobact.go directly mirrors mobile_activity() dispatch with per-section line references. But the concurrency model was never ported. C was single-threaded per pulse; Go spawns goroutines (AI ticker at 10s, zone dispatcher per-zone at 100ms, point update at 30s) without per-mob or per-zone synchronization.

Object system (object.go, location.go) has thorough typed-location design with tagged union and validation. MigrateCustomData/GetSaveState is a clean bridge. ZoneDispatcher is well-structured with context cancel, atomic counters, proper Stop() with WaitGroup — but runZoneMobAI is unimplemented.

The spawner (spawner.go) has deadlock-avoidance but tracking maps use flat VNum keys with pointer equality, which doesn't correctly track instance limits across zone resets.
