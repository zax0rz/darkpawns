## LLM-Level Analysis Findings

### HIGH-001: `doMemoryHookWithRetry` sends empty body on first HTTP attempt → DP-162
**File:** pkg/agent/memory_hooks.go (~line 80-100)
**What:** The retry wrapper reads + closes req.Body before entering the loop but only recreates it on retries (attempt > 0).
**Why it matters:** First attempt sends empty body; if REM Python returns 4xx (empty body = bad request), the <500 check treats it as success and returns nil. Memory events silently drop — kills, deaths, social interactions never reach Python for emotional tagging or REM synthesis.
**Source:** analysis (race/retry logic bug)
**Suggested fix:** Move body recreation out of `if attempt > 0` guard — always set req.Body before each attempt.

### MED-036: cmd/server/main.go goroutines not tracked for shutdown → DP-161
**File:** cmd/server/main.go (~line 181, ~line 223)
**What:** Zone reset goroutine and HTTP server goroutine spawned via `go func()` without WaitGroup tracking.
**Why it matters:** On SIGINT/SIGTERM, SaveWorld() races with in-flight zone reset. World state can corrupt on concurrent save.
**Source:** analysis (race/goroutine management)
**Suggested fix:** sync.WaitGroup tracking + context cancellation for graceful drain.
