# Security Audit — 2026-05-16 (Program 5)

## CRITICAL

**CRIT-001: Admin login unreachable — AuthMiddleware blanket-covers /admin/login**
- **File:** cmd/server/main.go:212, web/auth.go:26-54
- **What:** `http.Handle("/admin/", web.AuthMiddleware(adminRouter))` applies JWT auth middleware to ALL /admin/ paths including /admin/login. AuthMiddleware has no exception or skip-path for the login endpoint. The login endpoint is the only path that generates tokens — without it, no one can obtain a JWT.
- **Why it matters:** Admin panel is completely unusable. Chicken-and-egg deadlock: login requires a token, token requires login. Every request to /admin/login returns 401 before reaching the login handler.
- **Source:** adversarial analysis — auth middleware coverage audit
- **Suggested fix:** Register /admin/login as an exact-pattern handler on the default mux BEFORE the /admin/ subtree, same pattern used for /admin/health:
  ```go
  http.HandleFunc("/admin/login", adminRouter.ServeHTTP) // or a dedicated handler
  http.Handle("/admin/", web.AuthMiddleware(adminRouter))
  ```
  Or add a skip-paths list to AuthMiddleware.

## HIGH

**HIGH-001: Admin zone reset leaks internal error messages to builders**
- **File:** pkg/admin/handlers.go ~line 1976 (handleZoneResetTrigger)
- **What:** `http.Error(w, `{"error":"`+err.Error()+`"}`, http.StatusInternalServerError)` — raw Go error messages serialized into HTTP 500 responses. Any builder (level 33+) can probe invalid zone numbers and receive raw internal errors.
- **Why it matters:** Information disclosure — leaks implementation details, DB error strings, file paths to any player who reaches builder level. Attackers can map the internal error surface.
- **Source:** adversarial analysis — error exposure audit
- **Suggested fix:** Replace with `http.Error(w, `{"error":"internal server error"}`, http.StatusInternalServerError)`

## MEDIUM

**MED-001: luaDofile TOCTOU race condition in path validation**
- **File:** pkg/scripting/engine.go (luaDofile function)
- **What:** Path traversal check (Clean/Rel/prefix) happens before DoFile(). If the target file is replaced with a symlink between the check and the execution, directory traversal is possible despite the validation passing.
- **Why it matters:** Sandbox escape via TOCTOU — attacker who can time a symlink swap in the scripts directory can execute arbitrary Lua files outside the sandbox directory. Requires local filesystem access or compromised scripts dir.
- **Source:** adversarial analysis — sandbox escape audit
- **Suggested fix:** Open file first, validate the real path of the open file descriptor (e.g., via `/proc/self/fd/N` or os-specific resolution), then execute if path is within sandbox.

## LOW

**LOW-001: validateStringField misses Unicode control characters**
- **File:** pkg/admin/handlers.go:53-70
- **What:** Control character check loops over bytes 0x00-0x1F (ASCII control chars) but doesn't check for Unicode control characters (U+0080-U+009F, zero-width characters, bidi markers). Multi-byte runes containing Unicode control characters pass validation.
- **Why it matters:** Malicious Unicode content can be stored in admin-editable strings (zone names, room descriptions, mob keywords), potentially affecting output rendering or enabling injection in downstream consumers.
- **Source:** adversarial analysis — input validation audit
- **Suggested fix:** Add `unicode.IsControl(r)` and `unicode.Is(C, r)` checks for runes outside the ASCII range.
