### 1. NEW-003: executeCommand Stub → Full Implementation

**Files changed:** `pkg/game/damage_stubs.go`, `pkg/game/combat_control.go`, `pkg/game/world.go`, `pkg/session/manager.go`

The `doOrder` command now actually works. Previously it parsed, notified the room, said "Ok." and did nothing.

- **Player path:** `executeCommand` dispatches through `World.CommandExec` callback to the session layer's command registry. Callback wired at boot in `NewManager`. Thread-safe (`GetSession` uses `RLock`, `cmdRegistry.Lookup` uses `RLock`).
- **Mob path:** New `executeMobCommandForMob` delegates to existing `executeMobCommand` World method. `doOrder` now handles specific player targets, specific mob targets, and the "followers" keyword (both player and mob followers).
- **New helpers:** `getMobRoomVis` (mob lookup by name in room), `GetMobFollowersInRoom` (mob followers of a leader).
- **Cross-package:** No circular imports. Bridge is via `CommandExecFunc` on `World`, same pattern as existing `MessageSinkFunc`/`CloseConnectionFunc`.
- **`doForced`** benefits from this change automatically (delegates to `executeCommand`).
